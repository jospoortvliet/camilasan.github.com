Ruby on Rails Workshop: Intro

Linux

  • Install RVM (not as root!): $ \curl -sSL https://get.rvm.io | bash -s stable --rails
  • Edit your .bashrc and add: [[ -s "$HOME/.rvm/scripts/rvm" ]] && source "$HOME/.rvm/scripts/rvm" # Load RVM into a shell session *as a function*
  • Reload/reopen your terminal.
  • Check your installation running: $ rvm --version
    $ ruby --version
    $ rails --version
  • The default database for Rails is sqlite3. So you need to check if you have it installed: $ sqlite3 --version Otherwise, please, run: $zypper in sqlite-devel
  • Then if all goes well you can create a Rails project: $ rails new blah
    $ cd blah
    $ rails s
    # runs the server     If there are no error messages and you can see the welcome message at http://localhost:3000, then you are good to start to code :)
  • There is a common error that you can get when you run rails server: /~/username/.rvm/gems/ruby-2.1.1/gems/execjs-2.0.2/lib/execjs/runtimes.rb:51:in `autodetect': Could not find a JavaScript runtime. See https://github.com/sstephenson/execjs for a list of available runtimes. (ExecJS::RuntimeUnavailable) Possible solution: In the root for your project, there is a file called Gemfile. This file contains the list of Gems that are being used in your project. Add the Gem 'therubyracer' to this list: gem 'therubyracer' Run bundle install and then rails server should be working now. $ bundle install
    $ rails s     More info at https://github.com/sstephenson/execjs/issues/71.
  • For coding you can use any text editor you like. I suggest Sublime Text: http://www.sublimetext.com/2. It has a good support for Ruby and Rails.

More info at http://rvm.io
